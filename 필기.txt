* Unreal Framework
*	- GameMode : 각 레벨의 규칙 설정, 레벨별로 설정 가능
*	- Actor : 레벨에 배치되는 모든 오브젝트가 상속 받는 클래스
*	- Pawn : Controller로 조종할 수 있는 특수한 액터 (Actor를 상속받았음)
*	- Charactor : Pawn에 휴머노이드 특성을 더 추가한 클래스 (Pawn을 상속 받았음)
*	- Controller : 폰에 행동을 지시할 수 있게 해주는 클래스. Pawn에 Controller를 붙이는 것을 빙의(Possess)라고 함

* 조명 (Light)
*	- 디렉셔널 라이트 (Dirctional Light)
*		- 방향만 있는 조명
*		- 게임 월드에 전역적으로 한 방향에서 빛이 들어와야 될 때 사용하는 조명
*		- 예시) 태양광
*	- 포인트 라이트 (Point Light)
*		- 한 점에서 특정 반경 내에 빛을 뿌리는 조명
*		- 거리가 멀어질수록 빛이 약해진다.
*		- 예시) 횃불, 폭발, 전구 등 작은 광원.
*	- 스포트 라이트 (Spot light)
*		- 한 점에서 특정 방향으로 원뿔 모양으로 빛을 뿌리는 조명
*		- 원뿔의 중심부에서 멀어질수록 빛이 약해진다.
*		- 예시) 무대 조명, 자동차 헤드라이트, 손전등 등
*
*	- 렉트 라이트 (Rect Light)
*		- 직사각형 면에서 빛이 나오는 조명
*		- 예시) 형광등, 네온사인, TV화면, 창문 등
*	- 스카이 라이트 (Sky Light)
*		- 하늘의 밝기나 색을 전체 씬에 골고루 적용하는 환경광
*		- 예시) 실외 배경, 전역적인 간접광

* 머티리얼 (Material)
*	- 3D 모델의 표면에 적용되어 색상, 질감, 빛 반사 등 시각적 특성을 결정하는 데이터
*
* 물리 기반 랜더링 / PBR(Physically Based Rendering)
*	- 실제 세계의 빛과 재질이 상호 작용하는 방식을 물리 법칙에 기반하여 시뮬레이션하는 랜더링 기법
*	- 직관적이고 자연스럽다.
*	- 주요 속성
*		- Base Color : 물체의 기본 색상
*		- Metalic : 금속성 정도 (0이면 비금속, 1이면 금속)
*		- Roughness : 표먼의 거칠기 (0이면 매끄러움, 1이면 거침)
*		- Normal : 표면의 법선 벡터

* 좌표계 
*	- 위치를 표시하기 위한 방법
*	- x, y, z 값으로 3차원 3D 공간의 위치를 표시
*	- 왼손 좌표계, 오른손 좌표계
*	- 3D 그래픽스에서 일반적으로 왼손 좌표계를 사용한다
*		- 언리얼에서는 회전된 왼손 좌표계를 사용
*			- x : 앞쪽(Forward), y : 오른쪽(Right), z : 위쪽(Up)
*			- x축, y축 회전은 반시계 방향이 +
*			- z축 회전은 시계 방향이 +
* 로컬 좌표계
*	- 액터의 원점을 기준으로 한 좌표계
* 월드 좌표계
*	- 월드의 원점을 기준으로 한 좌표계

* 벡터
*	- 힘의 크기와 방향을 표현

* 언리얼 블루프린트의 코드 분류 방법 
*	- 이벤트
*		- 리턴값이 없다
*		- 재사용성은 떨어짐 (로컬 변수 없음)
*	- 함수
*		- 리턴 값이 있음
*		- 딜레이는 안됨
*	- 매크로
*		- 리턴 값이 있음
*		- 딜레이도 됨
*		- 로컬 변수 없음 (이벤트 내용을 정리하는 느낌)

* 언리얼의 회전
*	- Yaw : Z축 회전
*	- Pitch : Y축 회전
*	- Roll : X축 회전
*
* 오일러 앵글 (Euler Angle)
*	- 회전을 x, y, z축이 각각 얼마씩 회전했느냐로 표현
*
* 짐벌락
*	- 오일러 앵글을 사용할 대 하나의 축이 잠기는 현상
*
* 쿼터니언
*	- 숫자 4개로 회전을 표현하는 방법
*	- 장점
*		- 계산이 빠르다
*		- 메모리도 적게 차지한다
*	- 단점
*		- 사람이 알아보기 힘들다

* 보간
*	- 시작 값과 마지막 값이 있을 때 중간 값을 구하는 것
